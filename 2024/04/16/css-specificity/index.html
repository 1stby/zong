CSS 權重--學習筆記

John Doe
2024-04-16

前言

在前端開發中，CSS扮演著非常重要的角色。它負責網頁的樣式呈現，讓網頁變得美觀和易於使用。然而當我們的樣式表變得越來越龐大和複雜時，難免會遇到一些樣式衝突的問題。這時候瞭解CSS權重的運作機制就變得非常重要。本文將深入探討CSS權重的基本概念、計算規則以及一些使用技巧，幫助你更好地掌控樣式，寫出高質量的CSS代碼。

CSS權重的基本規則

CSS權重是瀏覽器用來決定使用哪個樣式的規則。

當我們都想對一個元素，使用多種的樣式，這時就要看誰的權重高。

以下是CSS選擇器的優先順序：

行內樣式(Inline styles) > ID選擇器(#id) > 類選擇器(.class) > 元素選擇器(element) > *

在這個例子中，.nav-link的顏色會是紅色,因為類選擇器的權重高於元素選擇器。

如果權重相同那就會看先後順序：

nav-link的顏色會是綠色，權重相同，越後面越優先。

權重計算

為了能更準確的計算CSS權重，我將選擇器附上一個值，方便我們好算出誰的權重高。

  • Inline styles : 1000
  • #id : 100
  • .class : 10
  • element : 1


    • 特別注意!important的優先度是最高的（如同德州撲克的AA，可以覆蓋其他對子），但是再開發中盡量避免使用!important，因為它會破壞樣式表的自然層疊順序，使得調試和維護變得困難。

    CSS權重是前端開發中非常重要的一個概念。瞭解權重的計算規則和層疊機制，可以幫助我們更好地控制樣式，寫出高質量的CSS代碼。在實際開發中，我們應該儘量遵循最佳實踐，合理使用選擇器，避免過度依賴!important，並且適當利用繼承和模組化來提高代碼的可維護性。
